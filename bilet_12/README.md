### **1. Що таке автоматизоване тестування? Які його переваги та недоліки порівняно з ручним тестуванням?**

**Автоматизоване тестування** — це перевірка ПЗ за допомогою тестових скриптів/інструментів, які запускаються автоматично та порівнюють фактичний результат з очікуваним.

Переваги (порівняно з ручним):
- швидке повторення тестів (особливо регресія);
- стабільність і менше людських помилок;
- можна запускати після кожного коміту (CI);
- економія часу на довгій дистанції.

Недоліки:
- треба час на написання/підтримку тестів;
- не все можна легко автоматизувати (UI/візуальні перевірки);
- тести можуть “ламатися” через зміни в коді/інтерфейсі;
- потрібні навички та інструменти.
-------------------------------------------------------------------
### **2. Які інструменти IDE (наприклад, VS Code) допомагають у написанні та налагодженні тестів?**

Для роботи з тестами в IDE (наприклад, VS Code) використовуються такі інструменти:

- **Python Extension** — підтримка unittest/pytest, автодоповнення тестів, запуск тестів з IDE.
- **Test Explorer (Testing Panel)** — перегляд списку тестів, запуск окремих тестів та аналіз результатів.
- **Debugger** — налагодження тестів за допомогою брейкпоінтів і покрокового виконання.
- **IntelliSense** — підказки для методів перевірок (assertEqual, assertRaises тощо).
- **Інтегрований термінал** — запуск тестів командами python test.py.

### **3. Поясніть принцип «Defect Clustering» (Скупчення дефектів).**

**Defect Clustering** — принцип, що більшість дефектів зосереджена в невеликій частині модулів (умовно: 80% багів у 20% коду).

Що з цього випливає:
- найпроблемніші модулі треба тестувати інтенсивніше;
- варто аналізувати статистику багів і покривати тести саме “гарячі” місця;
- фокус тестування дає більше ефекту, ніж рівномірне “по всьому”.

### **4. Завдання: У файлі main.py створіть функцію find_min(numbers: list), яка повертає мінімальне число зі списку. Тест: У файлі test.py протестуйте функцію на списку з цілими та дробовими числами.**

### Умова:
1. У файлі `main.py` необхідно створити функцію `find_min(numbers: list)`, яка повертає мінімальне число зі списку.

2. У файлі `test.py` потрібно реалізувати unit-тести за допомогою стандартної бібліотеки `unittest`, які перевіряють коректність роботи функції на:
- списку цілих чисел;
- списку дробових чисел;
- списку з цілих і дробових чисел.

------------------------------------------------------------------------------------------------

### **Реалізація функції (`main.py`)**
```python
def find_min(numbers: list):
    """
    Повертає мінімальне число зі списку.
    """
    return min(numbers)
```
-------------------------------------------------------------------------------------------------
### **Реалізація unit-тестів (`test.py`)**

```python
import unittest
from main import find_min


class TestFindMin(unittest.TestCase):

    def test_min_integers(self):
        numbers = [5, 2, 9, -1, 0]
        self.assertEqual(find_min(numbers), -1)

    def test_min_floats(self):
        numbers = [2.5, 1.1, 3.9, -0.4]
        self.assertEqual(find_min(numbers), -0.4)

    def test_min_mixed(self):
        numbers = [3, 2.2, -5, 0.0, 10]
        self.assertEqual(find_min(numbers), -5)


if __name__ == "__main__":
    unittest.main()
```
